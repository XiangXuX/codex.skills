# Workflow

## Goal

Convert a vague travel request into a decision-ready plan and, when requested, into a polished report.

## Three-Stage Workflow

### Stage 1: Explore

Use when the user is still open-ended.

Output:
- 2-4 viable trip directions
- Short explanation of why each fits
- Clear tradeoffs
- A small number of follow-up choices

Do not:
- Over-research prices before direction is chosen
- Dump a full itinerary too early

### Stage 2: Converge

Use when the user starts preferring one direction.

Output:
- A locked planning brief
- Main constraints
- Main exclusions
- Decision rationale

### Stage 3: Deliver

Use when the brief is stable enough to act on.

Output:
- Bookable options
- Final itinerary
- Budget summary
- HTML/PDF report if requested

## Quality Bar

- The plan must be actually doable.
- The recommendation must explain why it wins.
- Expensive or time-sensitive claims must be sourced.
- If dates, nights, and routes conflict, resolve that before delivery.

# Routing Rules

## Core Objective

Recommend the plan with the best balance of:
- Executability
- Time efficiency
- Experience quality
- Cost control

## Default Heuristics

- Prefer same-airport car return unless open-jaw meaningfully improves the trip.
- Prefer fewer hotel changes unless the added stop has strong experience payoff.
- Avoid forcing a landmark if it breaks the overall route.
- Keep long travel legs away from late arrival and departure days when possible.
- Treat tours, ferries, and timed departures as anchor points.

## Pace Control

Default to steady rather than maximal.

Red flags:
- Too many one-night stays
- More than one major timed activity on the same day
- Ferry plus long drive plus flight on the same day
- High-risk hikes in a trip that otherwise prioritizes stability

## Decision Language

When comparing plans, explicitly say:
- smoother
- cheaper
- more scenic
- more tiring
- higher risk
- better value
